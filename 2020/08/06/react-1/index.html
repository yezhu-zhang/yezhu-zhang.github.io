1.1 创建一个react组件

react组件的后缀名可以是js或者jsx
react组件分类: 类分类(class组件)+函数式组件
类组件-es6 中类 class
函数式组件 - js中的函数(一半的函数,箭头函数)
写react其中写的就是js +react的语法{}
组件的首字母必须大写(class类的首字母要大写)
组件的调用就像是html标签一样,如果组件的首字母小写,就会当作html标签使用,但是html默认标签没有该标签,就不会显示
react分为两类:react+react-dom
任何形式的react组件,都必须导入react模块

1.2 创建一个类组件

App.jsX

import React from 'react'

class App extends React.Component {
  constructor (props) { // js 中含有的  构造器   -- react
    super(props)
  }

  render () { // 继承父类的函数 render () --- 渲染  --- 渲染是一个组件的结构

    // 使用()包裹 jsx 代码 
    // 虚拟DOM
    return (
      <div>
        hello react
      </div>
    )
  }
}

export default App

1.4 状态

提到状态 - 类组件 - js的class
定义一个类的基本属性 — 借助于类的构造函数 — js
react 加一个规范,初始化数据 this.state = {}

import React from 'react'

class App extends React.Component {
  constructor (props) { // js 中含有的  构造器   -- react
    super(props)
    this.state = {
        msg: 'hello react'// 初始化的数据 - 状态
    }
  }

  render () { 
    return (
      <div>
        hello react
        { this.state.msg }
      </div>
    )
  }
}

export default App

1.5 列表循环

边遍历边循环

import React from 'react'

class App extends React.Component {
  constructor (props) {
    super(props)
    // 初始化的数据 - 状态
    this.state = {
      list: ['aaa', 'bbb', 'ccc', 'ddd']
    }
  }

  render () {
    return (
      <div>
        <h1>react - 列表循环</h1>
        <ul>
          {
            // 边遍历 边渲染
            // 不能使用 for 循环（语法通不过） forEach（没有效果）
            // map 自带返回值
            this.state.list.map((item, index) => {
              return (
                <li key= { index }> { item } </li>
              )
            })
            // for () 一些就报错
            // this.state.list.forEach((item, index) => {
            //   return (
            //   <li key = { index }>{ item}</li>
            //   )
            // })
          }
        </ul>
      </div>
    )
  }
}

export default App

先遍历后渲染

import React from 'react'

class App extends React.Component {
  constructor (props) {
    super(props)
    // 初始化的数据 - 状态
    this.state = {
      list: ['aaa', 'bbb', 'ccc', 'ddd']
    }
  }

  render () {
    // 先遍历后渲染 -- 借助于 数组的 特性 [ <h1></h1>, <h1></h1>, <App />]
    const arr = []
    // for(var i =0; i< len; i++) {}
    // map
    // for (var i in list) {}
    // for (var item of list) {}
    this.state.list.forEach((item, index) => {
      arr.push(<li key={index}>{item}</li>)
    })
    return (
      <div>
        <h1>react - 列表循环1</h1>
        <ul>
          {
            arr
          }
        </ul>
      </div>
    )
  }
}

export default App

1.执行setState()之后干了什么？
一个大的任务需要 10个小目标必须完成，每完成一个，将其压入队列，直到多有的都已结束，回报任务完成

setState()方法通过一个队列机制实现state更新，当执行setState()的时候，会将需要更新的state合并之后放入状态队列，而不会立即更新this.state(可以和浏览器的事件队列类比)。如果我们不使用setState而是使用this.state.key来修改，将不会触发组件的re-render。如果将this.state赋值给一个新的对象引用，那么其他不在对象上的state将不会被放入状态队列中，当下次调用setState()并对状态队列进行合并时，直接造成了state丢失。
